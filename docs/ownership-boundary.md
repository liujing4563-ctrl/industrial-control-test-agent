# 职责边界

## 平台核心

负责人：系统负责人

负责公共架构、运行时、策略、执行器、证据、Schema、MCP 网关和系统集成。

## PLC I/O 领域

负责人：PLC I/O 领域负责人

负责 PLC 信号、点表、互锁、故障判据、模拟器业务规则和 Capability Pack 专业内容。

## MCU 领域

负责人：MCU 测试领域负责人

负责 MCU 接口、UART/GPIO/I²C 测试方法、故障判据、模拟器业务规则和 Capability Pack 专业内容。

## 审核规则

任何影响以下内容的修改，必须由对应领域负责人审核：

- 信号语义；
- 测试步骤；
- 故障原因；
- 诊断规则；
- 恢复方法；
- 硬件参数；
- 场景标准答案。

平台负责人可以维护格式、Schema、框架和公共接口，但不能独自批准领域专业内容。
