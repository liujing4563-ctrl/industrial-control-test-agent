# MCU UART 通信测试 — 操作说明

## 测试目标

验证 MCU 嵌入式控制板的 UART 通信功能是否正常：
1. 心跳检测 — 确认 MCU 在线
2. UART 帧发送与接收 — 验证数据完整性
3. 波特率匹配 — 检测通信参数配置
4. GPIO 辅助验证 — 辅助判断硬件状态

## 测试流程

1. 读取 MCU 心跳包 → 确认设备在线
2. 发送标准 UART 测试帧
3. 读取 MCU 返回帧 → 比对一致性
4. 读取相关 GPIO 状态
5. 如有异常，触发故障诊断子流程

## 故障场景

| 故障 | 症状 | 可能原因 |
|------|------|---------|
| 心跳超时 | read_heartbeat 无响应 | MCU 掉电/复位/连接断开 |
| 波特率不匹配 | UART 帧乱码 | 收发波特率配置不一致 |
| 帧丢失 | 发送帧无回应 | 缓冲区溢出/中断冲突 |
| GPIO 异常 | GPIO 电平不符合预期 | 硬件短路/引脚配置错误 |

## 前置条件

- MCU 已上电
- UART 连接线缆正常
- 测试环境供电稳定
